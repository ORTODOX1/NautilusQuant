# NautilusQuant v2 — Ортогональная Спиральная Матрица

## Исправленная модель: Многомерный торнадо вместо воронки

---

## КРИТИЧЕСКОЕ ИСПРАВЛЕНИЕ (v2)

**Ошибка v1:** Центростремительное масштабирование `φ^(-i/d)` ИЗМЕНЯЛО длину вектора (радиус), что ЛОМАЛО скалярные произведения в attention-механизме. Нейросеть работает на `dot(q, k)` — если изменить `|k|`, результат attention будет неправильным.

**Исправление v2:** Золотое сечение и π применяются ТОЛЬКО к углам вращения. Радиус (длина вектора) НЕ ТРОГАЕТСЯ. Матрица преобразования строго **ортогональна**: `T^T · T = I`.

**Метафора:** Не воронка, засасывающая к центру, а **многомерный торнадо** — перекручивает точки по пропорциям ракушки, но не меняет их удалённость от центра координат.

---

## 1. Фундаментальная идея (исправленная)

NautilusQuant строит **детерминированную ортогональную матрицу вращения**, где углы между осями подчиняются золотому сечению, а не случайному генератору.

Ключевые принципы:
- **Радиус НЕ трогаем** → `|T·v| = |v|` для любого вектора v (норма сохраняется)
- **Углы закручиваем по φ** → золотой угол 137.5° обеспечивает оптимальное покрытие
- **Матрица ортогональна** → `T^T · T = I` (обратное преобразование тривиально)
- **Детерминирована** → seed не нужен, результат всегда один и тот же

## 2. Математика: Ортогональная Спиральная Матрица

### 2.1. Построение матрицы

Ортогональная матрица размера d×d строится как произведение d/2 плоских вращений (вращений Гивенса), где каждый угол определяется золотым сечением:

```
T = G(0,1,θ₀) · G(2,3,θ₁) · G(4,5,θ₂) · ... · G(d-2,d-1,θ_{d/2-1})
```

Где `G(i,j,θ)` — матрица вращения Гивенса в плоскости (i,j) на угол θ:

```
G(i,j,θ) = I  (единичная матрица)
  с заменой:
  G[i][i] = cos(θ)
  G[i][j] = -sin(θ)
  G[j][i] = sin(θ)
  G[j][j] = cos(θ)
```

### 2.2. Формула углов — Золотая спираль

Вместо случайных углов (TurboQuant) используем ряд на основе золотого угла:

```
θₖ = (2π / φ²) × (k + 1) = 137.507764° × (k + 1)

где k = 0, 1, 2, ..., d/2 - 1 (номер пары координат)
    φ = (1 + √5) / 2 ≈ 1.6180339887
```

### 2.3. Многослойная закрутка (Butterfly Spiral)

Для полного перемешивания координат одного прохода недостаточно. Применяем несколько «слоёв» вращений со сдвигом пар:

```
Слой 1: Пары (0,1), (2,3), (4,5), ...          углы: θ₀, θ₁, θ₂, ...
Слой 2: Пары (1,2), (3,4), (5,6), ...          углы: θ₀·φ, θ₁·φ, θ₂·φ, ...
Слой 3: Пары (0,3), (1,4), (2,5), ...          углы: θ₀·φ², θ₁·φ², θ₂·φ², ...
```

Каждый слой — произведение вращений Гивенса. Произведение ортогональных матриц = ортогональная матрица.

```
T = Layer₃ · Layer₂ · Layer₁

T^T · T = Layer₁^T · Layer₂^T · Layer₃^T · Layer₃ · Layer₂ · Layer₁ = I  ✓
```

### 2.4. Почему это сохраняет радиус

Для любого вектора v:
```
|T · v|² = (T·v)^T · (T·v) = v^T · T^T · T · v = v^T · I · v = |v|²  ✓
```

Длина вектора **не меняется**. Скалярное произведение **сохраняется**:
```
dot(T·q, T·k) = (T·q)^T · (T·k) = q^T · T^T · T · k = q^T · k = dot(q, k)  ✓
```

Attention-механизм работает **идентично** оригиналу.

## 3. Полный пайплайн (исправленный)

### Вход: KV-кэш вектор
```
v = [v₀, v₁, ..., v_{d-1}] ∈ FP16, |v| = R (радиус сохраняется!)
```

### Шаг 1: Ортогональное спиральное вращение

```python
def nautilus_rotate(v, dim):
    """Ортогональное вращение с золотыми углами. |v| НЕ меняется."""
    PHI = (1 + sqrt(5)) / 2
    golden_angle = 2 * pi / (PHI ** 2)  # ≈ 2.3999 рад
    result = v.copy()

    # Слой 1: соседние пары
    for k in range(dim // 2):
        i, j = 2*k, 2*k + 1
        theta = golden_angle * (k + 1)
        c, s = cos(theta), sin(theta)
        a, b = result[i], result[j]
        result[i] = a * c - b * s  # ТОЛЬКО вращение
        result[j] = a * s + b * c  # БЕЗ масштабирования!

    # Слой 2: сдвинутые пары (перемешивание)
    for k in range(dim // 2 - 1):
        i, j = 2*k + 1, 2*k + 2
        theta = golden_angle * (k + 1) * PHI
        c, s = cos(theta), sin(theta)
        a, b = result[i], result[j]
        result[i] = a * c - b * s
        result[j] = a * s + b * c

    # Слой 3: дальние пары (butterfly)
    stride = dim // 4
    for k in range(dim // 2):
        i = k
        j = (k + stride) % dim
        if i == j: continue
        theta = golden_angle * (k + 1) * PHI * PHI
        c, s = cos(theta), sin(theta)
        a, b = result[i], result[j]
        result[i] = a * c - b * s
        result[j] = a * s + b * c

    return result  # |result| == |v|  ✓
```

**Ключевое отличие от v1:** Нет `scale = φ^(-i/d)`. Только `cos(θ)` и `sin(θ)`. Вращение Гивенса по определению ортогонально.

### Шаг 2: Перевод в полярные координаты
```
r_k = √(v'_{2k}² + v'_{2k+1}²)     // радиус пары (НЕ равен |v|, это проекция!)
α_k = atan2(v'_{2k+1}, v'_{2k})     // угол пары
```

Благодаря золотым углам вращения, распределение α_k **сконцентрировано и предсказуемо**.

### Шаг 3: Скалярное квантование Lloyd-Max
```
levels = 2^b
q = round((val − min) / (max − min) × (levels − 1))
```

Overhead = 0 бит (углы матрицы вычисляются из формулы, а не хранятся).

### Шаг 4: QJL коррекция (1 бит)
```
sign_bit = sign(original − quantized)
corrected = quantized + sign_bit × |error| × 0.5
```

### Результат
```
Бит/значение: b + 1 (квант + QJL)
Overhead: 0 бит (матрица детерминирована)
Сжатие: 16 / (b+1) при b=3 → 4x, при b=2 → 5.3x
|T·v| = |v|  ✓  (радиус сохранён)
dot(T·q, T·k) = dot(q, k)  ✓  (attention не сломан)
```

## 4. Сравнение v1 vs v2

| | v1 (НЕПРАВИЛЬНО) | v2 (ПРАВИЛЬНО) |
|---|---|---|
| Масштабирование радиуса | φ^(-i/d) — ЛОМАЕТ dot product | Нет — радиус сохранён |
| Матрица | НЕ ортогональна | Строго ортогональна: T^T·T=I |
| Что делает | Воронка (сплющивает к центру) | Торнадо (крутит без сжатия) |
| Норма вектора | Изменяется ❌ | Сохраняется ✓ |
| dot(q,k) после преобразования | Искажён ❌ | Точен ✓ |
| Годится для attention | НЕТ | ДА ✓ |

## 5. Почему золотые углы лучше случайных (при ортогональности)

Золотой угол 137.507764° — «самый иррациональный» угол:
- Цепная дробь φ = [1; 1, 1, 1, ...] сходится медленнее всех
- По теореме о трёх расстояниях: N точек на окружности с шагом 2π/φ² дают ≤3 различных расстояния, стремящихся к равным
- Случайные углы дают O(1/√N) равномерность, золотой угол даёт O(1/N)

В контексте квантизации:
- Лучшая равномерность углов → более предсказуемое распределение после полярного преобразования
- Предсказуемое распределение → меньшая ошибка Lloyd-Max квантизатора
- Меньшая ошибка → лучшее качество при том же количестве бит

## 6. Визуальная метафора

**TurboQuant:** Берёте кубик Рубика и случайно крутите грани. Точки перемешиваются, но без гарантии равномерности.

**NautilusQuant v2:** Берёте кубик и крутите каждую грань на золотой угол. Как торнадо — всё вращается по спирали ракушки, но ни один кусочек не приблизился к центру и не удалился от него. Расстояния сохранены, но ориентация оптимизирована.
